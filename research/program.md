# Cross-Reference Research

## Цель

Улучшать покрытие и качество параллельных мест, не смешивая автоматические предложения с отредактированным корпусом.

## Границы

- Можно менять `data/json/cross-refs/candidates.json` и файлы внутри `research/`.
- Можно добавлять служебные скрипты в `data/scripts/`, если они помогают проверять кандидатов.
- Не менять `data/json/cross-refs/manual.json` без отдельного редакторского решения.
- Не менять тексты в `data/json/bible/books/` в рамках исследования параллелей.

## Формат кандидата

Записи в `candidates.json` используют тот же формат, что и `manual.json`:

```json
{
  "from": "1kgs:3:5",
  "to": "2ch:1:7",
  "toEnd": 12,
  "theme": "same_event",
  "note": "Описание предполагаемой параллели"
}
```

Разрешённые темы: `same_event`, `fulfillment`, `prophecy`, `theological`, `genealogy`, `tsk`.

## Цикл работы

1. Выбрать узкий фрагмент: книгу, главу или уже известную связку книг.
2. Добавить только проверяемые кандидаты в `candidates.json`.
3. Запустить `bun run validate:crossrefs`.
4. Убрать всё, что не проходит структурную проверку или дублирует `manual.json`.
5. Оставить короткую заметку в `note`, почему связь предложена.

## Критерий готовности

Кандидаты готовы к человеческому ревью, когда `bun run validate:crossrefs` проходит без ошибок.
